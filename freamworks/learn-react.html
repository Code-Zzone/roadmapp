دورة تعلم React.js

1. مقدمة في React.js

  • ما هو React.js؟: هو مكتبة JavaScript لبناء واجهات المستخدم التفاعلية باستخدام مكونات صغيرة.
  • لماذا React؟: لأنها تتيح لك بناء واجهات تفاعلية بشكل سريع وأداء عالي باستخدام Virtual DOM.
  • تركيب React: يتم تركيب React باستخدام npm أو عن طريق أدوات مثل Create React App.

2. المكونات (Components)

  • المكونات في React: يتم بناء واجهات المستخدم في React باستخدام مكونات (Components).
  • مكونات دالة (Functional Components): هي مكونات تكتب كدوال بسيطة وتُستخدم بشكل أساسي في React.
  • المكونات الكائنية (Class Components): هي مكونات يمكن أن تحتوي على حالة (State) ويمكنها استخدام طرق الحياة الخاصة بالمكون.

3. JSX

  • ما هو JSX؟: هو امتداد للـ JavaScript يسمح لك بكتابة HTML داخل JavaScript، مما يجعل الكود أكثر وضوحًا.
  • استخدام JSX: في JSX، يجب أن يكون الكود داخل أقواس، وتُستخدم علامات HTML عادة مثل <div> و <h1>.
  • الفرق بين JSX و HTML: في JSX، لا يمكنك استخدام بعض الخصائص مثل `class` بل تستخدم `className` بدلاً منها.

4. الحالة (State)

  • ما هي الحالة (State)؟: هي مجموعة من البيانات التي تتحكم في مظهر المكون وسلوكه.
  • تحديث الحالة: يمكنك تحديث الحالة باستخدام دالة setState().
  • التعامل مع البيانات داخل المكونات: الحالة تمثل القيم المتغيرة التي تعتمد عليها الواجهة.

5. الأحداث (Events)

  • التعامل مع الأحداث في React: React يوفر طريقة لالتقاط الأحداث مثل النقر على الأزرار باستخدام دوال مثل onClick.
  • التعامل مع الإدخالات: يمكنك ربط المدخلات بـ state باستخدام onChange.
  • الفرق بين الأحداث في React و JavaScript التقليدي: في React، يتم استخدام camelCase للأحداث مثل onClick و onSubmit.

6. التوجيه (Routing) باستخدام React Router

  • React Router: مكتبة تُستخدم لإنشاء التوجيه بين الصفحات في تطبيقات React.
  • أساسيات التوجيه: يمكنك تحديد المسارات باستخدام <Route> و <Link> للتنقل بين الصفحات.
  • التوجيه الديناميكي: يمكنك استخدام متغيرات في المسار للحصول على بيانات ديناميكية مثل معرّفات المستخدمين.

7. إدارة الحالة (State Management)

  • إدارة الحالة المحلية: يمكن إدارة الحالة داخل المكونات باستخدام useState (في المكونات الدالة).
  • إدارة الحالة العالمية: يمكن استخدام مكتبات مثل **Redux** لإدارة حالة التطبيقات الكبيرة والمعقدة.
  • Context API: تقدم React حلًا مدمجًا لإدارة الحالة عبر مكونات متعددة باستخدام **Context API**.

8. مكونات الوظائف (Hooks)

  • ما هي Hooks؟: هي دوال تسمح باستخدام حالة (State) ومؤثرات جانبية (Side Effects) في المكونات الدالة.
  • استخدام useState(): لتخزين البيانات المتغيرة داخل المكون.
  • استخدام useEffect(): لتنفيذ التعليمات البرمجية التي تؤثر على المكون بعد التحديثات.

9. تحسين الأداء

  • التقليل من عمليات إعادة الرندر: من خلال استخدام React.memo لتجنب إعادة الرندر غير الضرورية للمكونات.
  • تأجيل تحميل البيانات (Lazy Loading): تحميل المكونات بشكل غير متزامن لتحسين الأداء باستخدام React.lazy().
  • تجنب إنشاء دوال جديدة داخل JSX: لتقليل التكلفة الزائدة في إعادة الرندر.

10. أدوات تطوير React

  • React Developer Tools: هي إضافة للمتصفح تساعدك في تتبع الحالة والمكونات أثناء التطوير.
  • Create React App: هو أداة جاهزة لبدء مشاريع React بسهولة.
  • Testing Libraries: مثل **Jest** و **React Testing Library** لاختبار المكونات.
